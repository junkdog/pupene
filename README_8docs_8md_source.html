2 ## pupene
3 
4 A non-invasive, multi-format serialization library for C++17.
5 
6 
7 ### Features or Caveats
8 
9 This library was primarily conceived as a foundation for assisting with building
10 tooling for gamedev projects. The approach and naming conventions are borrowed
11 from Daniel Randle's [Pupping - a method for serializing data][article].
12 
13 - Works with default constructible types.
14  - `pupene` relies on direct field access to read/write.
15 - Concise and simple:
16  - `pup()` - one serializer-agnostic read/write function per type.
17  - `Pupper` - serializer, binary and optional JSON support included.
18  - Serialization format can not be customized, this makes it non-ideal for
19  dealing with third-party formats.
20  - Zero dependencies, unless built with JSON support.
21 - Not limited to performing only pure serialization.
22  - Example using [imgui and pupene][imgui_example] to auto-generate editor UI:s at runtime.
23 
24  [imgui_example]: https://github.com/junkdog/pupene_example
25  [article]: https://www.gamedev.net/articles/programming/general-and-gameplay-programming/pupping-a-method-for-serializing-data-r4485/

35 ### A brief introduction
36 
37 At the core, `pupene` is built around serializers called _puppers_, and `pup`
38 functions - a combined read/write per serializable type.
39 
40 Including `<pupene/pupene.h>` gets everything except JSON support, which
41 is done with `<pupene/json.h>` - when built with `PUPENE_BUILD_JSON=true`.
42 
43 Refer to the [API docs](https://junkdog.github.io/pupene) for more detailed
44 documentation. The [tests][tests] may be of some interest too.
45 
46 Namespace `pupene` is omitted from example code, in the interest of brevity.
47 
48  [tests]: https://github.com/junkdog/pupene/tree/master/test

51 #### Pupper
52 
53 `Pupper` is on the receiving end of `pup`; they only deal directly with values once
54 `pup` has flattened the object into a sequence of integer, decimal and string types.
55 
56 Included Puppers, and related convenience functions:
57 - JsonReader: `from_json()`
58 - JsonWriter: `to_json()`
59 - BinaryReader: `from_binary()`
60 - BinaryWriter: `to_binary()`
61 - DebugPupper: `to_debug()`
62 
63 
64 A `pup` function must exist for each serializable type.
65 
66 #### pup
67 
68 An object is serializable when there's a matching `pup` function; these describe
69 the structure of the type by enumerating its fields. All `pup` functions must
70 be defined in the `pupene::fns` namespace.
71 
72 A limited number of fundamental types are built-in:
73 - Integer and decimal types
74 - `std::string`
75 - STL containers, such as `std::vector` and `std::map`
76 
77 `pup()` for a type `FooBar` has the following signature:
78 
79 ```{.cpp}
80 template <typename P>
81 void pup(Pupper<P>& p, // Pupper-agnostic - works with all
82  FooBar& value, // must be reference
83  const Meta& meta); // name and Meta::Type
84 ```
85

87 #### Example
88 
89 You have some models:
90 
91 ```{.cpp}
92 struct Color {
93  float r, g, b, a;
94 };
95 
96 struct FontReference {
97  std::string path;
98  Color inner;
99  Color outer;
100 };
101 ```
102 
103 To make the types eligible, each requires its own pup function in the namespace `pupene:fns`.
104 
105 ```{.cpp}
106 namespace pupene::fns { // required namespace
107 
108  template<typename P>
109  void pup(Pupper<P>& p, // the serializer in use
110  Color& color, // always as non-const references
111  const Meta& meta) { // metadata for color
112 
113  // idiomatic boilerplate
114  pup_object(p, color, meta, [&color](auto&& fpup) {
115  fpup(color.r, "r"); // enumerate each field
116  fpup(color.g, "g");
117  fpup(color.b, "b");
118  fpup(color.a, "a");
119  });
120  }
121 
122  template<typename P>
123  void pup(Pupper<P>& p,
124  FontReference& ref,
125  const Meta& meta) {
126 
127  pup_object(p, ref, meta, [&ref](auto&& fpup) {
128  fpup(ref.path, "path");
129  fpup(ref.inner, "inner"); // dispatches to pup for Color
130  fpup(ref.outer, "outer");
131  });
132  }
133 }
134 ```
135 
136 Once the necessary pup:s are implemented, serialization between different
137 format becomes possible e.g.:
138 
139 ```{.cpp}
140 // load binary from input stream
141 std::istream& in = ...;
142 Color color = from_binary<Color>(in);
143 
144 // save to json (assumes built with PUPENE_BUILD_JSON=true)
145 std::string raw_json = to_json(color);
146 ```
147 
148 `Color` and `FontReference` can now also be combined with some STL containers:
149 
150 ```{.cpp}
151 std::unordered_map<Color, FontReference> colored_fonts = {
152  {colorA, small}, {colorB, medium}, {colorC, large}
153 };
154 
155 // complex keys - such as Color - are ok
156 std::string raw_json = to_json(colored_fonts);
157 
158 // (this doesn't require any user-supplied pup:s)
159 std::vector<int> numbers = {0, 1, 6, -2, 4, 5, 7};
160 to_binary(numbers, std::cout);
161 ```

164 ### Building
165 
166 Set up project:
167 
168 ```{.bash}
169 cmake -H. -B_build \
170  -DCMAKE_INSTALL_PREFIX=_install \
171  -DCMAKE_BUILD_TYPE=Release \
172  -DPUPENE_BUILD_TESTS=true
173 ```
174 
175 Build project:
176 ```{.bash}
177 cmake --build _build
178 ```
179 
180 Tests are run with:
181 
182 ```{.bash}
183 cmake --build _build --target test
184 ```
185 
186 Tests can also run with valgrind/memcheck:
187 ```{.bash}
188 cmake --build _build --target valgrind
189 ```

191 ### Using
192 
193 #### Hunter
194 
195 [Hunter](https://github.com/ruslo/hunter) is a package manager for CMake.
196 
197 ```{.cmake}
198 ### hunter init, top of CMakeLists.txt
199 include("cmake/HunterGate.cmake")
200 HunterGate(
201  URL "https://github.com/ruslo/hunter/archive/v0.19.99.tar.gz"
202  SHA1 "55ca271e9276b9973176e3f5a8c0d0dc25e46a53"
203  LOCAL # optional, will use cmake/Hunter/config.cmake
204 )
205 
206 ### adding pupene
207 hunter_add_package(pupene)
208 find_package(pupene CONFIG REQUIRED)
209 
210 ### ... later, when linking
211 target_link_libraries(example_project PUBLIC pupene::pupene)
212 
213 ```
214 
215 Build-time configuration can be customized in `cmake/Hunter/config.cmake`, for example:
216 
217 ```{.cmake}
218 hunter_config(
219  pupene
220  CMAKE_ARGS
221  PUPENE_BUILD_JSON=true
222 )
223 ```
